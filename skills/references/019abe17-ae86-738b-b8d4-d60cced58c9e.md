# 凡是给offer的公司，面试时基本不问技术细节，那些问得又多又细的公司，后面基本就没下文了。

## 详细内容

凡是给offer的公司，面试时基本不问技术细节，那些问得又多又细的公司，后面基本就没下文了。
有的面试你讲得头头是道，对方听得也津津有味，最后却悄无声息地没下文，这是为什么？因为有些公司，压根就不是想真招人，他们要的只是你的思路、解决方案和免费咨询。比如：“你上一家公司是怎么做服务拆分的？”“你们怎么处理数据库高并发的？”“能不能讲讲你们灰度发布的方案？”“有没有做过异地多活？流程是怎么控制的？”你兴冲冲地把以前公司几年的经验全盘托出，还想着对方听了肯定觉得我牛，结果聊完冷淡收场，连个 HR 通知都懒得发。你以为你在展示能力，其实你在白送干货。要知道，现在很多公司嘴上说招人，心里想着省钱，他们就靠面试来偷学解决方案，反正聊聊又不花钱。你分享得越细，对方越偷得踏实。这种情况现在非常多，尤其在行情冷、技术团队缺核心的时候更明显。所以，如果你在面试中发现对方一直在深挖你过往公司的完整解决方案，却对你项目的其他部分没什么兴趣，十有八九就是来白嫖思路的。这时候该收就得收，别什么都讲太细，别当冤大头。所以，问得多问得细，不一定是想招你，你可能只是解决方案提供者，或者 HR 完成 KPI 的一部分。
反过来看那些问得不多、甚至聊得很轻松就过了的公司，往往是这样的情况：岗位急招人；老板亲自上阵，知道要找怎样的人；团队比较小，技术方向不复杂，只要能干活就行；或者团队已经被人走了一批，再不招人就要跪。这些公司的面试流程更务实，他们关注的是：“你有没有做过类似的项目？”“你来之后能不能马上上手？”“你是不是个能沟通、能合作、不内耗的人？”技术细节？懂不懂 Java 虚拟机的内存模型？有没有 JVM 的优化思路？对他们来说真的没那么重要。所以面试就像一次聊天，重点不在于考你，而在于判断你和他们能不能快速配合起来做事。
