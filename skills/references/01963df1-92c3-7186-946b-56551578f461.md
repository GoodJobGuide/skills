## 详细内容

#### 现在面试到底看重的是什么呢？



从事java开发五年多了，但是技术确实不是很好，目前也是在疯狂恶补中，面试了好多家了，都没有合适的，我开始怀疑到底是不是自己的技术问题呢？有的公司的面试，感觉回答上来百分之七八十了，剩下一部分，确实不太了解，自己说的也比较诚恳，但是就是没有后续结果，都说不合适，我觉得应该是自己技术的问题，但是面试了这么多家公司，很少有问到比较重复的面试题，几乎问的都不太一样，也不知道该怎么准备了。
可能是自己表达能力的问题吧，感觉写代码没问题，但是说出来的话，可能表达的不是那么清楚，而且好多东西其实并不是很常用到的，所以也有些遗忘了，哎，只能继续背八股文了，虽然进入公司之后，CRUD的概率很大，但是没办法。
我真的搞不懂，为什么有的公司，一上来就问你一些概念的意义，某家公司，我说完自己的期望薪资之后，上来问我封装和多态的意义是什么，有什么优点。还有的公司，因为有朋友在，所以也知道公司的技术水平，但是面试官问的往往比这高大上很多，哎，真的太难了，有没有比较不错的公司推荐呢？

### 评论 2

看以往的工作经历跟当下公司契合不契合，这是其一，其二是通过面试问问题看看简历有没有造假、夸大的成分。举个例子，比方说自己主导个一个项目，那就要问你并发量多大，多少个用户，有没有做负载均衡，服务器内存占用多少，用什么软件监控 jvm；如果对答如流，那就至少相信你简历上写的“主导” “独立负责”之类的没有夸大，如果没有答上来，那就说明自己不是主力程序员。如果你觉得你做的事就是 CURD，那不好意思，是因为你看到的业务只有 CURD，别的看不到而已；你可以自己写自己熟练使用 docker，那就要问问你 docker 使用过程中发现了 docker 有没有什么大坑；如果你说 没有挺好的 完全没有问题，按照官方配置咋用咋好使，那就说明 docker 用的不咋样；
